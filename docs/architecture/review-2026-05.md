# FileOctopus Architecture Review — May 2026

Status: review (not an ADR). Scope: full stack — Rust workspace (`crates/`), Tauri shell
(`apps/desktop-tauri`), frontend (`packages/`), and the IPC contract that joins them.

This document captures **issues, gaps, duplicates, and improvements**. Claims were spot-verified
against the source (line counts, prop counts, greps, and a compile/test pass); where an initial
finding turned out to be wrong on inspection it is called out explicitly so the team does not act on
a false premise. A set of low-risk fixes was implemented alongside this review — see
[Fixes applied](#fixes-applied-this-pass).

## 1. Summary & scorecard

The architecture is sound: a clear trust boundary at IPC, domain-only `vfs`, a planned-job pipeline,
and — notably — an **IPC contract that is genuinely in sync across all four layers** (Rust DTOs ↔
`commandMap` ↔ TS clients ↔ Tauri handlers): 76 commands, 41 error codes, and 18 event constants
all mirror with no drift. That is a real strength and the main thing to protect.

The weak spots are concentrated in (a) incomplete remote/cloud providers, (b) a few Rust ergonomics
issues, and (c) frontend state/composition sprawl.

| Area                                         | Maturity           | Headline                                         |
| -------------------------------------------- | ------------------ | ------------------------------------------------ |
| `vfs`, `app-ipc`, `app-core`, `config`       | Mature             | Solid; `app-ipc` is one 2380-line file           |
| `fs-core`                                    | Mature             | Local ops complete; traversal already cycle-safe |
| `jobs`                                       | Minimal → improved | Pause was a poll loop (now condvar)              |
| `remote-core`                                | Mature             | Type-erased sessions via `downcast_ref`          |
| `provider-sftp` / `-s3`                      | Working            | Read+write functional                            |
| `provider-smb`                               | Incomplete         | Browse works; several write ops stubbed          |
| `provider-gdrive` / `-dropbox` / `-onedrive` | Stub               | Read-only; no OAuth token refresh                |
| `git-intel`, `terminal-core`                 | Working            | Minimal surface                                  |
| `telemetry`, `platform`                      | Stub/minimal       | Reserved/placeholder                             |
| `plugin-core`                                | Stub               | Not yet implemented                              |
| IPC contract                                 | Excellent          | Fully mirrored, zero drift                       |
| Frontend state                               | Needs work         | 187-field shell context; oversized files         |

## 2. Issues (architectural smells)

### Rust

- **Type-erased provider dispatch.** All six remote providers obtain a session via
  `as_any().downcast_ref::<T>()`; a type mismatch degrades to a generic "invalid session handle"
  at runtime rather than being caught at compile time (`crates/provider-*/src/provider.rs`). The
  boilerplate was deduplicated this pass (see fixes); the runtime-typing trade-off remains inherent
  to the `Arc<dyn RemoteSession>` design and is acceptable, but worth a comment for future readers.
- **`jobs` pause was a poll loop.** `wait_while_paused` spun on `thread::sleep(50ms)`
  (`crates/jobs/src/lib.rs`). Resume latency was up to 50 ms and the thread woke needlessly.
  **Fixed** with a `Condvar` (instant resume; bounded cancel poll).
- **`app-ipc` was a single 2380-line file** holding ~150 DTOs. **Fixed**: the pure DTO definitions
  were moved into 17 per-domain modules (`fs`, `network`, `search`, `file_operations`, `common`, …),
  shrinking `lib.rs` to ~1111 lines (uses/event constants/`error_codes`/`IpcError`/conversions/helpers);
  the serde wire format and the `app_ipc::Foo` public paths are unchanged.

### Frontend

- **Shell-context prop explosion.** `ShellLayoutContextValue` exposes **187 fields**
  (`packages/frontend/src/shell/ShellLayoutContext.tsx`), spread as props through the shell subtree,
  so unrelated state changes re-render broadly. This is the single highest-value refactor: move
  dialog-open flags and handler bundles into focused contexts or an external store and pass only
  `client` + `dispatch` widely.
- **Oversized files mixing concerns** (rendering + menu + utilities): ~~`Sidebar.tsx` 1204~~
  (**context menus extracted this pass** → 704 lines + `contextMenus.tsx`),
  `FileOctopusApp.tsx` 943, ~~`MenuBar.tsx` 871~~ (**split** → 149 lines + `menus/*`),
  ~~`OperationDialogView.tsx` 829~~ (**job/op helpers extracted** → 607 + `operationJobState.ts`),
  `useAppInit.ts` 788,
  `TerminalProvider.tsx` 742, ~~`panelStore.ts` 664~~ (**helpers extracted** → 534 + `paneSort.ts` +
  `panelStorage.ts`), ~~`DialogOverlayGroup.tsx` 643~~ (**split** → 188 lines +
  `dialogOverlaySections.tsx` + `fallbackPreferences.ts`), ~~`FileTable.tsx` 662~~ (**sub-components
  extracted** → 507 + `FileTableParts.tsx`).
- **Wiring hooks do too much.** `useAppInit.ts` (788) and `useEventHandlers.ts` (399) each wire many
  unrelated event domains; splitting per domain (nav/jobs/prefs/network/fs) would make them testable.

### Cross-cutting

- **Unconditional `console.warn` on every client creation** (`packages/ts-api/src/client.ts`) — debug
  spam shipped to users. **Fixed** (removed).

## 3. Gaps (incomplete / stubbed)

- **Cloud providers are read-only** — `provider-gdrive/-dropbox/-onedrive` return
  `ProviderCapabilities::read_only()` and have **no OAuth token refresh**; long operations fail when
  a token expires. Either implement write+refresh or hide these providers in the UI until ready.
- **SMB write ops are partly stubbed** (`provider-smb`): browse/stat work; create/move/delete need
  completion.
- **Stub crates:** `plugin-core` (not implemented), `platform`/`telemetry` (placeholder/minimal). The
  embedded telemetry has no hot-reload of the `log_level` preference.
- **Half-wired frontend features:** git is a status badge only (no diff/branch UI); autostart is
  gettable but has no settable control surfaced; the plugin UI is fragmented across
  `SettingsPlugins` and `PluginManagerDialog`; operation history is display-only (no undo flow).
- **No job timeout/deadline** — a stalled remote op blocks until the user cancels.
- **Doc/reality drift:** `CLAUDE.md` describes `config` as an empty reserved crate, but it is a
  mature ~2675-line crate (preferences/profiles/migrations). Update the doc.

## 4. Duplicates

- **Remote-provider `session_for` boilerplate** — profile-id parse → `session_for_profile` →
  `downcast_ref` → typed error → clone — was repeated across all six providers. **Fixed**: extracted
  `ConnectionSessionManager::typed_session_for` in `remote-core`.
- **`spawn_blocking(...).await.map_err(VfsError::internal)??`** repeated across sftp/smb/s3 providers.
  **Fixed**: extracted `remote_core::run_blocking_io`, adopted at all 17 call sites.
- **Per-client `try/catch` + `normalizeIpcError`** was repeated in every `clients/*.ts`. **Fixed**:
  `normalizeTransport` wraps the transport once in the `FileOctopusClient` constructor; all 89
  per-method wrappers were removed (−273 net lines).

### Corrected non-findings (verified false on inspection)

- **`panelStore` / `state/slices` are NOT a duplicate reducer.** `panelStore.ts` delegates to the
  slices via `reducePanelAction` (`panelStore.ts:149`); the slices are a deliberate layered split.
  Do **not** "consolidate" them as duplication.
- **Recursive traversal is already symlink-cycle-safe.** Folder-size keeps a canonicalized `visited`
  set and skips symlinks (`crates/fs-core/src/metadata.rs:143,179`); search skips symlinks too
  (`crates/fs-core/src/search.rs:146`). No DoS guard is missing here — no change needed.

## 6. Fixes applied this pass

Low-risk, high-value, verified (`cargo test -p jobs`, clippy, rustfmt, ts-api typecheck+tests, eslint
all green):

1. **Removed user-facing `console.warn`** in `packages/ts-api/src/client.ts`.
2. **Condvar-based job pause** — `crates/jobs/src/lib.rs` `PauseToken` now uses `Mutex<bool>` +
   `Condvar`; resume wakes the waiter immediately, cancel is observed within a bounded poll. New test
   `pause_token_wait_resume_wakes_promptly` asserts prompt wake-up.
3. **`typed_session_for` helper** in `crates/remote-core/src/session.rs`, adopted by all six remote
   providers — removes the repeated `session_for_profile` + `downcast_ref` boilerplate behind one
   typed error path.
4. **`app-ipc` split into 16 domain modules** — DTOs grouped by domain; `lib.rs` 2380 → ~1154 lines;
   wire format and public paths unchanged (verified by `cargo check --workspace` + `cargo test -p app-ipc`).
5. **`run_blocking_io` helper** in `crates/remote-core/src/blocking.rs`, adopted at all 17
   `spawn_blocking` call sites in the sftp/smb/s3 providers — collapses the `.map_err(...)??` join
   boilerplate to `.await?`.
6. **Centralized TS client error normalization** — `normalizeTransport` wraps the transport once in
   `FileOctopusClient`; removed all 89 per-method `try/catch { throw normalizeIpcError(error) }`
   wrappers across `packages/ts-api/src/clients/*.ts` (−273 net lines; 50 tests still pass).
7. **`MenuBar.tsx` split** — extracted the 7 inline menu-item builders into `shell/menus/*` modules
   plus a shared `menus/types.ts` (`MenuBarProps`/`MenuId`/`MenuHelpers`/`menuShortcut`); `MenuBar.tsx`
   871 → 149 lines, public `MenuBarProps` re-exported unchanged (frontend suite: 2022 tests pass).
8. **`Sidebar.tsx` context menus extracted** — the four context-menu components moved to
   `sidebar/contextMenus.tsx`; `Sidebar.tsx` 1204 → 704 lines (frontend suite: 2022 tests pass).
9. **`OperationDialogView.tsx` helpers extracted** — the 12 pure job-snapshot/merge and
   name/uri/error helpers (used by 18 files) moved to `dialogs/operationJobState.ts` and re-exported
   for compatibility; view file 829 → 607 lines (frontend suite: 2022 tests pass).
10. **`DialogOverlayGroup.tsx` split** — the 245-line flat render of 18 dialogs broken into three
    pass-through section components in `dialogOverlaySections.tsx`, and the `FALLBACK_PREFERENCES`
    default literal moved to `fallbackPreferences.ts`; main file 643 → 188 lines (props interface +
    thin composition). `DialogOverlayGroupProps` unchanged (frontend suite: 2022 tests pass).
