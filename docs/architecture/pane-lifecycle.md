# Pane loading lifecycle

## Frontend state

Each pane tab tracks:

- `loadState`: `idle`, `loading`, `loaded`, `empty`, `error`, `permissionDenied`, or `timeout`
- `activeRequestId`: latest navigation generation identifier
- `sessionId`: backend listing stream identifier

Navigation clears entries and sets `loadState` to `loading`. Directory batches update entries and derive the terminal state from entry count and batch error codes.

## IPC contract

`fs.list_start` accepts:

- `requestId`: required correlation identifier generated by the frontend per navigation
- `panelId`: optional pane key used to cancel superseded listings for the same pane

The response echoes `sessionId` and `requestId`. Each `directory:batch` event includes both values. The frontend ignores batches when `requestId` does not match the tab's `activeRequestId`.

## Backend behavior

- Superseded listings for the same `panelId` are cancelled
- Listings are wrapped in a 30 second timeout and emit `timeout` on expiry
- Permission failures surface `permission_denied`
