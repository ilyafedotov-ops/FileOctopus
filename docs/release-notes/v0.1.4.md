# FileOctopus v0.1.4 Release Notes

FileOctopus v0.1.4 is a patch release that rolls the post-release
documentation and preview metadata updates into rebuilt desktop packages.

## Supported

- Local filesystem navigation through `local://` resource URIs.
- Two-panel file browsing with sorting, filtering, keyboard navigation, and
  virtualized large lists.
- Planned copy, move, rename, create-folder, create-file, trash, permanent
  delete, archive create/extract, and conflict-handling operations.
- Job progress, cancellation, pause/resume, operation history, startup
  interruption marking, and diagnostics export.
- Preferences for theme, density, view mode, hidden files, layout, operation
  behavior, terminal, network, editor, viewer, diagnostics, shortcuts, and
  autostart.
- Command palette, text preview, columns view, expanded table columns,
  filesystem watcher, navigation/favorites/starred IPC, hash and folder-size
  helpers, content search, and checksum verification.
- Git status and review, embedded local/SSH terminal, SFTP/SMB/S3 profiles,
  cloud provider connector crates, plugin marketplace, ACL editor, diff/merge,
  sync directories, tags/labels, saved searches, media/PDF/text previews,
  vertical split, and storage gauge.

## Changes Since v0.1.3

- Updated README, build, and architecture references to the current release.
- Published current release notes in-repo before rebuilding packages.
- Aligned browser-preview app info and diagnostics fixtures with version
  `0.1.4`.
- Rebuilt release assets for macOS Apple Silicon, macOS Intel, Windows, and
  Linux from the current `main` branch.

## Platforms

- GitHub release assets are published for macOS Apple Silicon, macOS Intel,
  Windows x64, and Linux x64.
- Platform-specific smoke testing is still required for Trash, symlinks, locked
  files, long paths, terminal shells, and native installer behavior.

## Known Limitations

- Rubber-band selection, EXIF metadata display, trash browser/restore, AI
  semantic search, and peer-to-peer sync are deferred.
- Symlink object copy is intentionally unsupported and fails with
  `unsupported_symlink`.
- Locked-file semantics are platform-specific; unsupported behavior must be
  recorded during QA.
- Diagnostics redacts the home directory path where practical but may include
  operation filenames from history.

## Bug Reports

Attach the diagnostics bundle from **Help -> Diagnostics** and include platform,
package type, filesystem location, and reproduction steps.
